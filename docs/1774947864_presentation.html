Core Concept

Three ways to analyse contracts

1

Analyze Document

Upload any PDF. Ask questions about it. Run a compliance check. The file is never stored — extracted, analysed, deleted.

POST /api/analyze · /api/compliance
2

Compare with DB

Upload a draft contract. System finds the most similar stored contracts and lets GPT-4o compare and contrast.

POST /api/analyze (mode=compare)
3

Search Knowledge Base

Ask anything about your stored contract portfolio. LangGraph agent routes, retrieves, reasons, cites sources.

POST /api/query (SSE stream)

🔒 Critical invariant: Temporary uploaded docs are NEVER written to ChromaDB — enforced at the route level, not the UI.

Split at Q: markers.
One chunk = one Q+A pair.
Preserves semantic unity.

Legal Contracts

Split at §, Article N, Clause N.
One chunk = one legal clause.
Atomic unit of legal meaning.

Q&A Ready

Questions they'll likely ask

Why RAG instead of fine-tuning the LLM on contract data?
New contracts queryable immediately — no retraining. Every answer cites source clauses (legal requirement). GDPR right-to-erasure: delete a vector store entry, not retrain the model. Fine-tuning bakes contract facts into model weights permanently.
Why LangGraph instead of a single LLM call?
State-machine graph makes each node independently testable. The query router node lets us handle "find missing clauses" differently from "find a specific clause" — document-grouped context vs flat retrieval. Future nodes (reranker, hallucination check) can be added without refactoring.
Why SSE instead of WebSocket?
SSE is strictly server→client — perfect for one question, one streamed answer. Works through all corporate firewalls as plain HTTP. No ping/pong or reconnect logic. FastAPI's StreamingResponse supports it natively. WebSocket adds bidirectional complexity with zero benefit here.
How do you prevent hallucination?
System prompt: "Answer only from the provided contract excerpts." Similarity threshold (0.40) filters low-relevance chunks before GPT-4o sees them. Formatter node appends source citations with relevance scores — every claim is auditable. RAG grounds answers in retrieved text.
